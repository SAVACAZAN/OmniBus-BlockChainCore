To add a new locale:\+


\begin{DoxyItemize}
\item Choose a directory name for itChoose a directory name for it
\end{DoxyItemize}.


\begin{DoxyItemize}
\item Copy the content of an existing locale (e.\+g. {\ttfamily en-\/\+US}) into the newly created directory.
\item Replace the values in the content with yours file-\/by-\/file. Use \href{https://www.unicode.org/cldr/charts/32/summary/root.html}{\texttt{CLDR data}} as a point of reference which values to choose.
\end{DoxyItemize}

\end{DoxyItemize}— contains long date localizer functions used by {\ttfamily format} and {\ttfamily format\+Relative}.
\begin{DoxyItemize}
\item \`{}format\+Relative\`{}{\ttfamily format\+Relative}
\end{DoxyItemize}— relative date localizer function used by {\ttfamily format\+Relative}.
\begin{DoxyItemize}

\begin{DoxyItemize}
\item \`{}match\`{}{\ttfamily match}
\end{DoxyItemize}— contains functions to parse date values. Required by {\ttfamily parse}.
\begin{DoxyItemize}
\item \`{}options\`{}{\ttfamily options}

\hypertarget{md_frontend_2node__modules_2date-fns_2docs_2i18n_contribution_guide_autotoc_md6350}{}\doxysubsubsection{\texorpdfstring{Formatting localizers}{Formatting localizers}}\label{md_frontend_2node__modules_2date-fns_2docs_2i18n_contribution_guide_autotoc_md6350}
For some languages, there is a difference between "{}stand-\/alone"{} localizers and "{}formatting"{} localizers. "{}\+Stand-\/alone"{} means that the resulting value should make grammatical sense without context. "{}\+Formatting"{} means that the resulting value should be declined using the grammar rules of the language as if the value was a part of a date. For example, for languages with grammatical cases, the stand-\/alone month could be in the nominative case ("{}\+January"{}), and the formatting month could decline as a part of the phrase "{}1st of January"{}. In this case, use parameters {\ttfamily formatting\+Values} and {\ttfamily default\+Formatting\+Width} of {\ttfamily build\+Localize\+Fn} function.

Any localizer could be stand-\/alone and formatting. Check the CLDR chart for the unit to see if stand-\/alone and formatting values are different for a certain unit. If there\textquotesingle{}s no difference (usually it happens in languages without grammatical cases), parameters {\ttfamily formatting\+Values} and {\ttfamily default\+Formatting\+Width} are not needed.

In this example, in Russian language a stand-\/alone month is in the nominative case ("{}январь"{}), and formatting month is in the genitive case ("{}января"{} as in "{}1-\/е января"{}). Notice the different endings:\+
