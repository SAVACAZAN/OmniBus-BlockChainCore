\hypertarget{dir_32e5ff581a7a2789b29af98a276fc074_autotoc_md10052}{}\doxysubsection{\texorpdfstring{About Tauri}{About Tauri}}\label{dir_32e5ff581a7a2789b29af98a276fc074_autotoc_md10052}
Tauri is a polyglot and generic system that is very composable and allows engineers to make a wide variety of applications. It is used for building applications for Desktop Computers using a combination of Rust tools and HTML rendered in a Webview. Apps built with Tauri can ship with any number of pieces of an optional JS API /\+ Rust API so that webviews can control the system via message passing. In fact, developers can extend the default API with their own functionality and bridge the Webview and Rust-\/based backend easily.

Tauri apps can have custom menus and have tray-\/type interfaces. They can be updated, and are managed by the user\textquotesingle{}s operating system as expected. They are very small, because they use the system\textquotesingle{}s webview. They do not ship a runtime, since the final binary is compiled from rust. This makes the reversing of Tauri apps not a trivial task.\hypertarget{dir_32e5ff581a7a2789b29af98a276fc074_autotoc_md10053}{}\doxysubsection{\texorpdfstring{This module}{This module}}\label{dir_32e5ff581a7a2789b29af98a276fc074_autotoc_md10053}
This is a typescript library that creates {\ttfamily cjs} and {\ttfamily esm} Java\+Script endpoints for you to import into your Frontend framework so that the Webview can call and listen to backend activity. \doxylink{namespace_we}{We} also ship the pure typescript, because for some frameworks this is more optimal. It uses the message passing of webviews to their hosts.

To learn more about the details of how all of these pieces fit together, please consult this \href{https://github.com/tauri-apps/tauri/blob/dev/ARCHITECTURE.md}{\texttt{ARCHITECTURE.\+md}} document.\hypertarget{dir_32e5ff581a7a2789b29af98a276fc074_autotoc_md10054}{}\doxysubsection{\texorpdfstring{Installation}{Installation}}\label{dir_32e5ff581a7a2789b29af98a276fc074_autotoc_md10054}
