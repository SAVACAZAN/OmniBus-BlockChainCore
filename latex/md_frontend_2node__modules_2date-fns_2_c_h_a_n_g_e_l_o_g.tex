\begin{DoxyItemize}
\item {\bfseries{BREAKING}}:\+ now functions don\textquotesingle{}t accept string arguments, but only numbers or dates. When a string is passed, it will result in an unexpected result ({\ttfamily Invalid Date}, {\ttfamily NaN}, etc).

From now on a string should be parsed using {\ttfamily parse\+ISO} (ISO 8601) or {\ttfamily parse}.

In v1 we\textquotesingle{}ve used {\ttfamily new Date()} to parse strings, but it resulted in many hard-\/to-\/track bugs caused by inconsistencies in different browsers. To address that we\textquotesingle{}ve implemented our ISO 8601 parser but that made library to significantly grow in size. To prevent inevitable bugs and keep the library tiny, we made this trade-\/off.

See \href{https://blog.date-fns.org/post/we-cut-date-fns-v2-minimal-build-size-down-to-300-bytes-and-now-its-the-smallest-date-library-18f2nvh2z0yal}{\texttt{this post}} for more details.

\item as before, arguments expected to be {\ttfamily Date} are converted to {\ttfamily Date} using {\itshape date-\/fns\textquotesingle{}} {\ttfamily to\+Date} function;
\item arguments expected to be numbers are converted to integer numbers using our custom {\ttfamily to\+Integer} implementation (see \href{https://github.com/date-fns/date-fns/pull/765}{\texttt{\#765}});
\item arguments expected to be strings are converted to strings using Java\+Script\textquotesingle{}s {\ttfamily String} function;
