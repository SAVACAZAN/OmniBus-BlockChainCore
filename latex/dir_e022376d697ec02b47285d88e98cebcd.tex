A Java\+Script library for escaping CSS strings and identifiers while generating the shortest possible ASCII-\/only output.

This is a Java\+Script library for \href{https://mathiasbynens.be/notes/css-escapes}{\texttt{escaping text for use in CSS strings or identifiers}} while generating the shortest possible valid ASCII-\/only output. \href{https://mothereff.in/css-escapes}{\texttt{Here’s an online demo.}}

\href{https://mths.be/cssescape}{\texttt{A polyfill for the CSSOM {\ttfamily CSS.\+escape()} method is available in a separate repository.}} (In comparison, {\itshape cssesc} is much more powerful.)

This function takes a value and returns an escaped version of the value where any characters that are not printable ASCII symbols are escaped using the shortest possible (but valid) \href{https://mathiasbynens.be/notes/css-escapes}{\texttt{escape sequences for use in CSS strings or identifiers}}.


\begin{DoxyCode}{0}
\DoxyCodeLine{cssesc('Ich\ ♥\ Bücher');}
\DoxyCodeLine{//\ →\ 'Ich\ \(\backslash\)\(\backslash\)2665\ \ B\(\backslash\)\(\backslash\)FC\ cher'}
\DoxyCodeLine{}
\DoxyCodeLine{cssesc('foo\ 𝌆\ bar');}
\DoxyCodeLine{//\ →\ 'foo\ \(\backslash\)\(\backslash\)1D306\ \ bar'}

\end{DoxyCode}


By default, {\ttfamily cssesc} returns a string that can be used as part of a CSS string. If the target is a CSS identifier rather than a CSS string, use the {\ttfamily is\+Identifier:\+ true} setting (see below).

The optional {\ttfamily options} argument accepts an object with the following options:\+\hypertarget{dir_e022376d697ec02b47285d88e98cebcd_autotoc_md5922}{}\doxysubsubsubsection{\texorpdfstring{{\ttfamily is\+Identifier}}{{\ttfamily is\+Identifier}}}\label{dir_e022376d697ec02b47285d88e98cebcd_autotoc_md5922}
The default value for the {\ttfamily is\+Identifier} option is {\ttfamily false}. This means that the input text will be escaped for use in a CSS string literal. If you want to use the result as a CSS identifier instead (in a selector, for example), set this option to {\ttfamily true}.
