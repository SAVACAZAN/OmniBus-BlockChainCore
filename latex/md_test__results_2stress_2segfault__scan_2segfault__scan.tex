{\ttfamily get\+Latest\+Block} returns {\ttfamily Block} by value but {\ttfamily Block} contains {\ttfamily array\+\_\+list.\+Managed(\+Transaction)} (heap pointer) and slice fields ({\ttfamily hash}, {\ttfamily previous\+\_\+hash}). The returned struct is logically a borrow with no enforced lifetime. Every caller currently can race against mining.

Two acceptable fixes:\+
\begin{DoxyItemize}
\item {\bfseries{Option A}} (strict):\+ require caller to hold {\ttfamily self.\+mutex}; add {\ttfamily assert(self.\+mutex.\+try\+Lock() == false)} debug check at top.
\item {\bfseries{Option B}} (safe-\/default):\+ return {\ttfamily Block\+Snapshot} — a flat POD struct with copied scalars and {\ttfamily \+[64]\+u8} hash buffers; do the deep copy inside {\ttfamily get\+Latest\+Block} while holding the lock internally.
\end{DoxyItemize}

Option B is preferred for RPC — handlers can stay concise and any future caller is safe.
