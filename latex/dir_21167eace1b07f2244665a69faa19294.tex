ESM configs can be loaded with {\bfseries{async API only}}. Specifically {\ttfamily js} files in projects with {\ttfamily "{}type"{}:\+ "{}\doxylink{namespacemodule}{module}"{}} in {\ttfamily package.\+json} or {\ttfamily mjs} files.\hypertarget{dir_21167eace1b07f2244665a69faa19294_autotoc_md7311}{}\doxysubsection{\texorpdfstring{Difference to {\ttfamily cosmiconfig}}{Difference to {\ttfamily cosmiconfig}}}\label{dir_21167eace1b07f2244665a69faa19294_autotoc_md7311}
Lilconfig does not intend to be 100\% compatible with {\ttfamily cosmiconfig} but tries to mimic it where possible. The key difference is {\bfseries{no}} support for yaml files out of the box({\ttfamily lilconfig} attempts to parse files with no extension as JSON instead of YAML). You can still add the support for YAML files by providing a loader, see an exampleexample below.\hypertarget{dir_21167eace1b07f2244665a69faa19294_autotoc_md7312}{}\doxysubsubsection{\texorpdfstring{Options difference between the two.}{Options difference between the two.}}\label{dir_21167eace1b07f2244665a69faa19294_autotoc_md7312}
