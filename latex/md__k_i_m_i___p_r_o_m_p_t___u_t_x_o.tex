You are a senior Zig systems engineer reviewing and patching an L1 blockchain codebase written in Zig 0.\+15.\+2. The chain is called {\bfseries{Omni\+Bus}}. Your task is to convert balance reads from a RAM cache ({\ttfamily bc.\+balances}, a {\ttfamily String\+Hash\+Map}) to be sourced from the existing on-\/chain UTXO set ({\ttfamily bc.\+utxo\+\_\+set}), in two distinct phases.

You will produce two patch series:\+

{\bfseries{Phase B (immediate, low-\/risk):\+}} keep {\ttfamily bc.\+balances} as a hot cache but make every external balance read go through the UTXO set. RAM cache becomes a write-\/only mirror, never read by RPC /\+ P2P /\+ wallet code. Add an audit assertion that compares the two paths in debug builds and logs divergence in release builds.

{\bfseries{Phase C (later, full Bitcoin model):\+}} delete {\ttfamily bc.\+balances} entirely. All balance and nonce data lives in chainstate. Aligned with the Bitcoin-\/style storage refactor spec already in the repo at {\ttfamily \doxylink{_a_r_c_h___b_i_t_c_o_i_n___s_t_o_r_a_g_e_8md}{ARCH\+\_\+\+BITCOIN\+\_\+\+STORAGE.\+md}}.

So the UTXO set already mirrors every confirmed credit. What\textquotesingle{}s missing:\+


\begin{DoxyEnumerate}
\item UTXOs from inputs are {\bfseries{not}} spent — {\ttfamily spend\+UTXO} is implemented but never called from {\ttfamily apply\+Block}. Instead {\ttfamily bc.\+balances} is debited directly. UTXOSet therefore only grows; balances drift from the UTXO sum after the first transfer.
\item {\ttfamily get\+Address\+Balance} reads {\ttfamily bc.\+balances} rather than {\ttfamily utxo\+\_\+set.\+get\+Balance}.
\item No audit /\+ divergence detector exists.
\end{DoxyEnumerate}\hypertarget{md__k_i_m_i___p_r_o_m_p_t___u_t_x_o_autotoc_md73}{}\doxysection{\texorpdfstring{THE BUG WE\textquotesingle{}RE FIXING}{THE BUG WE\textquotesingle{}RE FIXING}}\label{md__k_i_m_i___p_r_o_m_p_t___u_t_x_o_autotoc_md73}
{\ttfamily bc.\+balances} has no contract that ties it to chain state. Anything that writes to it directly (without going through {\ttfamily apply\+Block} → proper TX handling) creates a phantom balance that vanishes on the next replay-\/from-\/scratch.

\doxylink{namespace_we}{We} just lost 51 of 54 testnet faucet recipient balances at restart because the faucet RPC writes {\ttfamily bc.\+balances} directly without emitting a chain TX. Replay rebuilt only what was on chain → faucet grants gone. (See {\ttfamily \doxylink{_a_r_c_h___b_i_t_c_o_i_n___s_t_o_r_a_g_e_8md}{ARCH\+\_\+\+BITCOIN\+\_\+\+STORAGE.\+md}} for the full incident write-\/up.)

The narrow fix in this prompt is {\bfseries{not}} to make faucet emit TXs (that\textquotesingle{}s a separate RPC change). It\textquotesingle{}s to make sure that when a phantom write happens again, the rich list, RPC, and wallet UI {\bfseries{see only what\textquotesingle{}s actually on chain}} — i.\+e. UTXO sums, not the divergent RAM cache.\hypertarget{md__k_i_m_i___p_r_o_m_p_t___u_t_x_o_autotoc_md74}{}\doxysection{\texorpdfstring{OMNIBUS-\/\+SPECIFIC CONSTRAINTS}{OMNIBUS-\/\+SPECIFIC CONSTRAINTS}}\label{md__k_i_m_i___p_r_o_m_p_t___u_t_x_o_autotoc_md74}

\item {\bfseries{Account model on-\/chain wire format.}} Block transactions today have one {\ttfamily from\+\_\+address} → one {\ttfamily to\+\_\+address} + amount + fee. They don\textquotesingle{}t yet carry vin/\+vout vectors. Phase B keeps this; Phase C designs the migration.

A single patch series with these changes:\+\hypertarget{md__k_i_m_i___p_r_o_m_p_t___u_t_x_o_autotoc_md76}{}\doxysubsection{\texorpdfstring{B.\+1 — Make {\ttfamily apply\+Block} spend the inputs}{B.\+1 — Make {\ttfamily apply\+Block} spend the inputs}}\label{md__k_i_m_i___p_r_o_m_p_t___u_t_x_o_autotoc_md76}
