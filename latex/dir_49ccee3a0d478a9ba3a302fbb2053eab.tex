Generates low-\/s deterministic-\/k RFC6979 ECDSA signature. Requries hash of message, which means you\textquotesingle{}ll need to do something like {\ttfamily \doxylink{namespacesha256}{sha256(message)}} before signing.

{\ttfamily extra\+Entropy:\+ true} enables hedged signatures. They incorporate extra randomness into RFC6979 (described in section 3.\+6), to provide additional protection against fault attacks. Check out blog post \href{https://paulmillr.com/posts/deterministic-signatures/}{\texttt{Deterministic signatures are not your friends}}. Even if their RNG is broken, they will fall back to determinism.

Default behavior {\ttfamily lowS:\+ true} prohibits signatures which have (sig.\+s \texorpdfstring{$>$}{>}= CURVE.\+n/\+2n) and is compatible with BTC/\+\+ETH. Setting {\ttfamily lowS:\+ false} allows to create malleable signatures, which is default openssl behavior. noble-\/secp256k1 v2 improves security and reduces attack surface. The goal of v2 is to provide minimum possible JS library which is safe and fast.


\begin{DoxyItemize}
\item Disable some features to ensure 4x smaller than v1, 4KB bundle size:\+
\begin{DoxyItemize}
\item The features are now a part of \href{https://github.com/paulmillr/noble-curves}{\texttt{noble-\/curves}}, {\bfseries{switch to curves if you need them}}. Curves is drop-\/in replacement.
\item DER encoding:\+ to\+DERHex, to\+DERRaw\+Bytes, signing /\+ verification of DER sigs
\item Schnorr signatures
\item Support for environments which don\textquotesingle{}t support bigint literals
\item Common.\+js support
\item Support for node.\+js 18 and older without shimshim
\end{DoxyItemize}
\end{DoxyItemize}
\begin{DoxyItemize}
\item Using {\ttfamily utils.\+precompute()} for non-\/base point
\end{DoxyItemize}

{\ttfamily get\+Public\+Key}
\begin{DoxyItemize}
\item now produce 33-\/byte compressed signatures by default
\item to use old behavior, which produced 65-\/byte uncompressed keys, set argument {\ttfamily is\+Compressed} to {\ttfamily false}:\+ {\ttfamily get\+Public\+Key(priv, false)}
\end{DoxyItemize}

{\ttfamily sign}
\begin{DoxyItemize}
\item is now sync; use {\ttfamily sign\+Async} for async version
\item now returns {\ttfamily Signature} instance with {\ttfamily \{ r, s, recovery \}} properties
\item {\ttfamily canonical} option was renamed to {\ttfamily lowS}
\item {\ttfamily recovered} option has been removed because recovery bit is always returned now
\item {\ttfamily der} option has been removed. There are 2 options:\+
\begin{DoxyEnumerate}
\item Use compact encoding:\+ {\ttfamily from\+Compact}, {\ttfamily to\+Compact\+Raw\+Bytes}, {\ttfamily to\+Compact\+Hex}. Compact encoding is simply a concatenation of 32-\/byte r and 32-\/byte s.
\item If you must use DER encoding, switch to noble-\/curves (see above).
\end{DoxyEnumerate}
\end{DoxyItemize}

{\ttfamily verify}
\begin{DoxyItemize}
\item {\ttfamily strict} option was renamed to {\ttfamily lowS}
\end{DoxyItemize}

{\ttfamily get\+Shared\+Secret}
\begin{DoxyItemize}
\item now produce 33-\/byte compressed signatures by default
\item to use old behavior, which produced 65-\/byte uncompressed keys, set argument {\ttfamily is\+Compressed} to {\ttfamily false}:\+ {\ttfamily get\+Shared\+Secret(a, b, false)}
\end{DoxyItemize}

{\ttfamily recover\+Public\+Key(msg, sig, rec)} was changed to {\ttfamily sig.\+recover\+Public\+Key(msg)}
\begin{DoxyItemize}
\item {\ttfamily number} type for private keys have been removed:\+ use {\ttfamily bigint} instead
\item {\ttfamily Point} (2d xy) has been changed to {\ttfamily Projective\+Point} (3d xyz)
\item {\ttfamily utils} were split into {\ttfamily utils} (same api as in noble-\/curves) and {\ttfamily etc} ({\ttfamily hmac\+Sha256\+Sync} and others)
\end{DoxyItemize}\hypertarget{dir_49ccee3a0d478a9ba3a302fbb2053eab_autotoc_md4717}{}\doxysubsection{\texorpdfstring{Contributing \& testing}{Contributing \& testing}}\label{dir_49ccee3a0d478a9ba3a302fbb2053eab_autotoc_md4717}
