The implementation is loosely based on cryptocoinjs/\+hdkey, which has MIT Licensewhich has MIT License.\hypertarget{dir_5a97cf55226c792cbe513e9a14530c9f_autotoc_md4834}{}\doxysubsection{\texorpdfstring{Warnings about BIP32}{Warnings about BIP32}}\label{dir_5a97cf55226c792cbe513e9a14530c9f_autotoc_md4834}
BIP32 is a bad standard. It would be great if we\textquotesingle{}ve had something better.


\begin{DoxyItemize}
\item Network IDs (different currencies) are taken from a single Git\+Hub document called SLIP-\/0044
\item There were new projects, which did not yet have SLIP. Exchanges added support of those projects to their cold wallets. Then after the projects were added to SLIP, the exchanges were required to re-\/generate their cold wallets -\/ a complicated task
\item BIP32 is unusable for many different elliptic curves. For example, ETH2 uses bls12-\/381 curve, and with bip32 54\% of generated keys would be invalid. So, they’re using much better BLS-\/only EIP-\/2333 as a replacement.
\item It’s easy to shoot yourself in foot with non-\/hardened keys, which could allow simple de-\/anonimization of all addresses
\end{DoxyItemize}\hypertarget{dir_5a97cf55226c792cbe513e9a14530c9f_autotoc_md4835}{}\doxysubsection{\texorpdfstring{Security}{Security}}\label{dir_5a97cf55226c792cbe513e9a14530c9f_autotoc_md4835}
