Request interceptors are executed in {\bfseries{reverse order}} (LIFO -\/ Last In, First Out). This means the {\itshape last} interceptor added is executed {\bfseries{first}}.

\begin{DoxyItemize}
\item {\ttfamily \{\}} -\/ serialize the value with JSON.\+stringify
\item {\ttfamily \+[]\+} -\/ unwrap the array-\/like object as separate fields with the same key

\item {\ttfamily visitor:\+ Function} -\/ user-\/defined visitor function that will be called recursively to serialize the data object to a {\ttfamily Form\+Data} object by following custom rules.
\item {\ttfamily dots:\+ boolean = false} -\/ use dot notation instead of brackets to serialize arrays and objects;
\item {\ttfamily meta\+Tokens:\+ boolean = true} -\/ add the special ending (e.\+g {\ttfamily user\{\}:\+ \textquotesingle{}\{"{}name"{}:\+ "{}John"{}\}\textquotesingle{}}) in the Form\+Data key. \begin{DoxyItemize}
\item {\ttfamily null} -\/ don\textquotesingle{}t add brackets ({\ttfamily arr:\+ 1}, {\ttfamily arr:\+ 2}, {\ttfamily arr:\+ 3})
\item {\ttfamily false}(default) -\/ add empty brackets ({\ttfamily arr\+[]\+:\+ 1}, {\ttfamily arr\+[]\+:\+ 2}, {\ttfamily arr\+[]\+:\+ 3})
\item {\ttfamily true} -\/ add brackets with indexes ({\ttfamily arr\+[0]\+:\+ 1}, {\ttfamily arr\+[1]\+:\+ 2}, {\ttfamily arr\+[2]\+:\+ 3})

Axios has its own {\ttfamily Axios\+Headers} class to manipulate headers using a Map-\/like API that guarantees caseless work. Although HTTP is case-\/insensitive in headers, Axios will retain the case of the original header for stylistic reasons and as a workaround when servers mistakenly consider the header\textquotesingle{}s case. An Axios\+Headers object instance can contain different types of internal values. that control setting and merging logic. The final headers object with string values is obtained by Axios by calling the {\ttfamily to\+JSON} method.

\begin{quote}
