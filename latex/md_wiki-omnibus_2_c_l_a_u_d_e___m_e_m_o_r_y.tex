{\bfseries{Problem}}:\+ Script was STOPPING all existing miners before starting new ones. When adding miners in batches, each batch would replace the previous ones instead of adding to them.

{\bfseries{Fix}}:\+ Changed lines 41-\/61 to make miners add INCREMENTALLY:\+
\begin{DoxyItemize}
\item Track how many miners are currently running:\+ {\ttfamily local running=\$(get\+\_\+running\+\_\+count)}
\item Start new miners after existing ones:\+ {\ttfamily local start\+\_\+id=\$((10 + running))}
\item Don\textquotesingle{}t stop existing miners -\/ just add more
\end{DoxyItemize}

{\bfseries{Result}}:\+ {\ttfamily bash add-\/miners-\/staggered.\+sh 100 10 5} now correctly adds 100 miners instead of replacing them.\hypertarget{md_wiki-omnibus_2_c_l_a_u_d_e___m_e_m_o_r_y_autotoc_md11174}{}\doxysubsection{\texorpdfstring{2. Run Script (run.\+sh)}{2. Run Script (run.\+sh)}}\label{md_wiki-omnibus_2_c_l_a_u_d_e___m_e_m_o_r_y_autotoc_md11174}

{\bfseries{Fix}}:\+ Added genesis miner startup before extra miners (lines 89-\/101):\+

\DoxyCodeLine{\ \ echo\ \$!\ >>\ .miners\_pids}
\DoxyCodeLine{\ \ sleep\ 0.02}

\end{DoxyCode}
