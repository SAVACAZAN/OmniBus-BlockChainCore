{\ttfamily main.\+zig} is the entry point that orchestrates all subsystems in sequence:\+


\begin{DoxyEnumerate}
\item {\bfseries{Single-\/instance lock}} — OS-\/level lock prevents two miners on the same machine
\item {\bfseries{CLI parsing}} ({\ttfamily cli.\+zig}) → mode (seed/\+miner), node-\/id, host, port, seed-\/host
\item {\bfseries{Vault reader}} ({\ttfamily vault\+\_\+reader.\+zig}) → reads mnemonic from Super\+Vault Named Pipe, env var, or dev default
\item {\bfseries{Database}} ({\ttfamily database.\+zig}) → loads/\+persists blockchain state from {\ttfamily omnibus-\/chain.\+dat}
\item {\bfseries{Genesis}} ({\ttfamily genesis.\+zig}) → initializes chain with official genesis block (mainnet or testnet config)
\item {\bfseries{Wallet}} ({\ttfamily wallet.\+zig}) → derives wallet from mnemonic, 5 PQ address domains via liboqs
\item {\bfseries{Mempool}} ({\ttfamily mempool.\+zig}) → FIFO transaction pool with size/\+time limits
\item {\bfseries{Consensus}} ({\ttfamily consensus.\+zig}) → PoW engine with difficulty adjustment
\item {\bfseries{Metachain + Shards}} ({\ttfamily metachain.\+zig}, {\ttfamily shard\+\_\+coordinator.\+zig}) → 4-\/shard architecture
\item {\bfseries{Subsystems}} — state trie, finality (Casper FFG), staking, governance, peer scoring, DNS registry, guardian
\item {\bfseries{P2P}} ({\ttfamily p2p.\+zig}) → TCP transport, peer discovery, knock-\/knock duplicate detection
\item {\bfseries{Web\+Socket}} ({\ttfamily ws\+\_\+server.\+zig}) → push events to React frontend on port 8334
\item {\bfseries{RPC}} ({\ttfamily rpc\+\_\+server.\+zig}) → JSON-\/\+RPC 2.\+0 HTTP server on port 8332 (separate thread)
\item {\bfseries{Mining loop}} — Sub\+Block engine (10 × 0.\+1s sub-\/blocks → 1 Key\+Block), then mine main block
