{\bfseries{Why 8 decimals matter:\+}} at 4 decimals, a 9279-\/SAT balance shows as {\ttfamily 0.\+0000} (looks empty). At 8 decimals it shows {\ttfamily 0.\+00000928} (clearly small but non-\/zero). Anything below 100 SAT is dust and can\textquotesingle{}t be sent anyway.\hypertarget{md_docs_2_c_h_a_i_n___c_o_n_s_t_a_n_t_s_autotoc_md4210}{}\doxysection{\texorpdfstring{Common gotchas}{Common gotchas}}\label{md_docs_2_c_h_a_i_n___c_o_n_s_t_a_n_t_s_autotoc_md4210}

\begin{DoxyEnumerate}
\item {\bfseries{DUST\+\_\+\+THRESHOLD applies to {\ttfamily amount}, not {\ttfamily fee}.}} A TX with {\ttfamily amount=99, fee=1} is rejected (dust amount). A TX with {\ttfamily amount=100, fee=1} is accepted.
\item {\bfseries{OP\+\_\+\+RETURN TXs}} can have {\ttfamily amount=0} AND {\ttfamily fee\texorpdfstring{$>$}{>}=1}. They\textquotesingle{}re metadata carriers, not payments — {\ttfamily to\+\_\+address} is set but ignored at validation level.
\item {\bfseries{Nonce per address.}} Every TX from address {\ttfamily A} must have {\ttfamily nonce = previous\+\_\+nonce + 1}. Mempool rejects with "{}nonce conflict"{} if you reuse or skip. Use {\ttfamily getnonce} before each TX or maintain a local counter that never decrements.
\item {\bfseries{{\ttfamily sendtransaction} always signs with the node\textquotesingle{}s primary wallet.}} If you want to send from a different address, use {\ttfamily sendrawtransaction} with a client-\/side signed TX blob (see {\ttfamily transaction.\+zig:\+Transaction} schema).
\item {\bfseries{{\ttfamily balance\+OMNI} from RPC is a STRING}} (e.\+g., {\ttfamily "{}0.\+100024230"{}}), not a number. Don\textquotesingle{}t {\ttfamily parse\+Float} it for display — show the string directly to keep precision.
\item {\bfseries{Faucet cooldown is per-\/\+IP, not per-\/account.}} Running 100 claims from one IP needs 60s × 100 = 100 min minimum. Multiple IPs (separate VPS) can parallelize. 
\end{DoxyEnumerate}
