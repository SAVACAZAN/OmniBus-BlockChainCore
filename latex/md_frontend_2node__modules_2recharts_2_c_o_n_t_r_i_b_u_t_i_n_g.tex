Storybook also has a great interface for adding tests. By default every story in storybook is a smoke test (rendering without error logs means the test passed). Additionally, it is possible to add actual tests as play functions with an assert to a story. This will often be easier than using React Testing Library, because the natural test debugging tool is Storybook itself. See for example {\ttfamily storybook/\+stories/\+\+Examples/\+cartesian/\+\+Reference\+Line/\+\+Reference\+Line\+If\+Overflow.\+stories.\+tsx}\hypertarget{md_frontend_2node__modules_2recharts_2_c_o_n_t_r_i_b_u_t_i_n_g_autotoc_md8579}{}\doxysubsection{\texorpdfstring{Performance tests}{Performance tests}}\label{md_frontend_2node__modules_2recharts_2_c_o_n_t_r_i_b_u_t_i_n_g_autotoc_md8579}
Recharts uses (\href{https://callstack.github.io/reassure/}{\texttt{https:\+/\+/\+callstack.\+github.\+io/\+reassure/\+}})\+[Reassure]\+ to run performance tests. Here is how to use:\+


\begin{DoxyEnumerate}
\item Before making code changes, run {\ttfamily npm run test:\+perf:\+baseline}. This will generate baseline data. You only do this once for each change.
\item Make your code changes now
\item Run {\ttfamily npm \doxylink{namespacetest}{test}:\+perf}. Reassure will run tests again, compare with the baseline from step 1. and print results to the console. If you wish you can also review the results in a markdown file in path {\ttfamily ./\+reassure/\+output.md}.
\item If you are satisfied with the performance impact then go ahead and commit! If you wish to make more changes go back to step 2.
\end{DoxyEnumerate}

Reassure will look for {\ttfamily \texorpdfstring{$\ast$}{*}.perf-\/test.\+tsx} files by default.

Also be aware that the performance tests might take a long time to run; up to several minutes.\hypertarget{md_frontend_2node__modules_2recharts_2_c_o_n_t_r_i_b_u_t_i_n_g_autotoc_md8580}{}\doxysection{\texorpdfstring{\label{_pr}%

{\bfseries{Working on your first Pull Request?}} You can learn how from this {\itshape free} series \href{https://app.egghead.io/playlists/how-to-contribute-to-an-open-source-project-on-github}{\texttt{How to Contribute to an Open Source Project on Git\+Hub}}

{\itshape Before} submitting a pull request, please make sure the following is done…


\begin{DoxyItemize}
\item Search \href{https://github.com/recharts/recharts/pulls}{\texttt{Git\+Hub}} for an open or closed Pull Request that relates to your submission. You don\textquotesingle{}t want to duplicate effort.
\item Fork the repo and create your branch from {\ttfamily master}.
\item If you have added functionality or changed existing functionality, be sure to add a test. Ideally a unit test for helper function, or a test that includes rendering with RTL.
\item If you\textquotesingle{}ve changed APIs, make sure that the stories in Storybook are working as expected.
\item Ensure the test suite passes ({\ttfamily npm run \doxylink{namespacetest}{test}}).
\item Make sure your code lints ({\ttfamily npm run lint}) -\/ we\textquotesingle{}ve done our best to make sure these rules match our internal linting guidelines.
\end{DoxyItemize}\hypertarget{md_frontend_2node__modules_2recharts_2_c_o_n_t_r_i_b_u_t_i_n_g_autotoc_md8581}{}\doxysection{\texorpdfstring{\label{_code}%
