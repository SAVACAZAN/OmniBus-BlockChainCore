Full report at `STATUS/SECURITY_AUDIT_2026-05-07.md`. Key findings:

CRIT-01: dns_registry.zig `signed_required` defaults `false` in `DnsRegistry.init()` — all DNS ops unauth by default on mainnet.
CRIT-02: pq_crypto.zig `generateKeyPairFromSeed` returns zeroed struct on liboqs failure — silent zero-key derivation.
HIGH-01: secp256k1.zig `isValidPrivateKey` uses early-return byte comparison — timing oracle on private key.
HIGH-02: pq_crypto.zig deterministic RNG global mutex never released on `OQS_SIG_new` failure paths.
HIGH-03: pq_crypto.zig `verify()` passes unchecked signature.len directly to liboqs C.
HIGH-04: p2p.zig `payload_len` from wire — enforcement before allocation needs runtime verification.
HIGH-05: mempool.zig RBF replacement does not verify new TX signature.

**Why:** signed_required defaulting false was discovered as the most dangerous mainnet gap — any attacker can steal any DNS name without a signature if the node operator forgets enableSigned(true).

**How to apply:** When reviewing DNS-related code, always check signed_required and fee_enforcement flags are set at startup in main.zig.
