Audit #14 — 2026-05-23. Files read: transaction.zig, state_trie.zig, guardian.zig,
mempool.zig, p2p.zig, rpc_server.zig, governance.zig, grid_engine.zig.

**Why:** Continued from #13 — 2 CRIT, ~14 HIGH still unresolved going into this pass.

**Key findings:**
- SEV-42 STILL OPEN (transaction.zig:641-642, `pq_omni_*` and `hybrid_q*` return true)
- SEV-50 STILL OPEN (state_trie.zig:93-104 HashMap non-deterministic)
- SEV-75 NEW HIGH: `ctx.bc.mempool` is a raw `ArrayList(Transaction)` (blockchain.zig:238);
  all RPC handlers call `.append(tx)` directly, bypassing the `Mempool` struct's
  `add()` (sig verify, fee check, dedup, size limit). ~20 affected RPC handlers.
- SEV-76 NEW MEDIUM: `P2PNode.checkRateLimit()` defined at p2p.zig:1963 but never
  called in the inbound message loop (line 2374 calls dispatchMessage directly). Same
  dead code as prior audits noted.
- SEV-77 NEW MEDIUM: mempool.zig has no per-sender TX limit; one address can fill
  entire 1 MB mempool with min-fee TXs.
- SEV-66 (grid cancel no-auth): CONFIRMED FIXED at GridRegistry level (line 170-175
  checks owner). Still bypassable via SEV-75 RPC path.
- receipt_manager.zig and compact_block.zig: FILES DO NOT EXIST in core/ — SEV-61,
  SEV-62, SEV-68 may be obsolete/resolved by removal.
- governance_onchain.zig vote() has double-vote guard via `self.voted` HashSet (good).
- governance.zig GovernanceEngine.vote() still has no dedup (SEV-57 still open).
- rpc_server.zig:16189 `tier` taken from JSON body, no on-chain validation (SEV-58 still open).
- rpc_server.zig:7819 still uses `std.mem.eql` not `constantTimeEql` (SEV-01/72 still open).
- guardian.zig:100-110 removeGuardian still no auth parameter (SEV-41 still open).
- block.zig calculateHash still does not include fills_root (SEV-43 still open).

**Total open:** 2 CRIT, 11 HIGH, 7 MEDIUM = 20 open findings.

**STATUS file write permission:** STILL BLOCKED (same as all prior audits).

**How to apply:** SEV-75 is the most impactful new finding — fixing it alone closes
the signature bypass loop for all ~20 RPC-submitted TXs. After SEV-42 and SEV-75,
the next most impactful is SEV-50 (consensus split risk).
