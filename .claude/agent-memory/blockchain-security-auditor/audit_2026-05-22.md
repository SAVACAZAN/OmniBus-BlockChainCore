### New Findings

- **SEV-66** HIGH: grid_create owner not authenticated — rpc_server.zig:14590 accepts owner string without secp256k1 sig verify; attacker can create grids on behalf of any address, spamming orderbook with 200 ghost orders per call (200 * 256 = 51,200 max orders)
- **SEV-67** MEDIUM: Grid price*amount u64 overflow — grid_engine.zig:406, `sell_p * amount_base / 1_000_000` silently wraps for large price*base values; use u128 intermediate
- **SEV-68** MEDIUM: Varint shift overflow — binary_codec.zig:32,40; `shift: u6` wraps at 64 after 10 continuation bytes; need max-10-bytes guard and explicit overflow check
- **SEV-69** MEDIUM: Self-trade manipulation of oracle price — matching_engine.zig allows self-trade (deliberate design), but no rate limit means 1000 self-trades/block saturates fill buffer and corrupts price history / oracle feed
- **SEV-70** HIGH: governance vote() caller-supplied power overflow — governance.zig:201; `voting_power: u64` not verified against on-chain stake; passing maxInt causes `total_voted_power` overflow making quorum_needed = 0
- **SEV-71** HIGH: key_encryption verifyPassword stub — key_encryption.zig:165; any 8-128 char password accepted as correct; `reencryptWithNewPassword` can be called by attacker to steal private key
- **SEV-72** HIGH: constantTimeEql defined but unused — rpc_server.zig:7819; one-line fix: `return constantTimeEql(got, token);`
- **SEV-73** MEDIUM: grid tick() no per-block rate limit — grid_engine.zig:383; if caller invokes tick multiple times per block, each call fills another level without HTLC settling
- **SEV-74** MEDIUM: order_swap_link lockTaker() no maker-locked verification — order_swap_link.zig:278; can transition pending->both_locked without maker HTLC being funded
