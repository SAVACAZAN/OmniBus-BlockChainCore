# Ninth Security Audit — 2026-05-14

**Why:** Full pass on 10 priority targets not deeply covered in prior passes (transaction.zig, block.zig, state_trie.zig, binary_codec.zig, archive_manager.zig, matching_engine.zig, order_swap_link.zig, price_oracle.zig) plus re-verification of known-open findings.

**How to apply:** New findings join the open tracking table. SEV-33 (previously attributed to pq_crypto.zig) is now confirmed located in transaction.zig:633-634 and tracked as SEV-42.

## New Findings This Pass

### SEV-42 — CRITICAL — transaction.zig:633-634
`Transaction.verifySignature()` unconditionally returns `true` for schemes `pq_omni_ml_dsa`, `pq_omni_falcon`, `pq_omni_dilithium`, `pq_omni_slh_dsa`, `hybrid_q1..q4`. Any forged PQ-OMNI or hybrid TX passes mempool signature verification. All 8 PQ-OMNI and hybrid address types are affected. Fix: implement proper per-scheme verification instead of `=> true`.

### SEV-43 — HIGH — block.zig:159-180
`Block.calculateHash()` commits to `merkle_root` and `prices_root` but NOT `fills_root`. A miner can alter trade fill records in a broadcast block without invalidating PoW. Fix: add `hasher.update(&self.fills_root)` in `calculateHash()`.

### SEV-44 — HIGH — price_oracle.zig:221-249
`DistributedPriceOracle.submitPrice()` never verifies the `signature` field on `MinerPriceSubmission`. Any P2P peer can submit arbitrary prices under spoofed miner addresses. Extends SEV-26 to the distributed oracle layer. Fix: verify ECDSA signature in submitPrice().

### SEV-45 — MEDIUM — price_oracle.zig:432-433
`isOutlierPrice()` computes `diff * 10000` on u64; overflows for diff > ~1.8e15 (possible with extreme price submissions). Same issue in `ConsensusPrice.deviationPct()` at line 74. Fix: use u128 intermediate.

### SEV-46 — MEDIUM — order_swap_link.zig:278-283
`lockTaker()` transitions binding to `.both_locked` regardless of whether `lockMaker()` was called. Taker can reach `.both_locked` + call `settle()` without maker actually locking funds. Fix: require both maker_locked + taker_locked flags before transitioning.

### SEV-47 — MEDIUM — state_trie.zig:93-105
`calculateRootHash()` iterates `std.AutoHashMap` (non-deterministic order), producing different root hashes for the same state on different runs. Consensus-critical. Fix: collect + sort account hashes before hashing.

### SEV-48 — HIGH — transaction.zig:569-636
Sub-component of SEV-42: the hash integrity check (`hash_bytes == expected_hash`) is also skipped for PQ-OMNI/hybrid schemes due to the unconditional `return true`.

### SEV-49 — MEDIUM — binary_codec.zig:106-120
`BinaryEncoder.encodeTransaction()` writes from_address and to_address as raw bytes without length prefix. Decoder cannot determine field boundaries. Fix: add 1-byte length prefix for each address.
